Libraries

Ideas for Angular libraries:

  • A library of UI components that can be used to build web applications. This could include components for things like buttons, forms, tables, and charts.
  • A library of utility functions that can be used to perform common tasks, such as loading data from an API, validating user input, and formatting text.
  • A library of services that can be used to provide access to common resources, such as a database, a file system, or an API.
  • A library of directives that can be used to add custom behavior to elements in the DOM.
  • A library of pipes that can be used to transform data before it is displayed to the user.



I created a library of utilities functions doing the following...

  1. 'ng g library random-utilities', this also adds a project type of library to app-wide angular.json
  2. Then I updated functions in random-utilties.service.ts...which surprise a library works like a service
  3. build the service for use using 'ng build random-utilities'
  4. npm i random-utilities
  5. In this libraries component I used import RandomUtilitiesService from ...; at the top to use library functions like a service
